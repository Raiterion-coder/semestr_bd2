# Управление спортивными данными с логированием

Полнофункциональное приложение для управления спортивными данными (Спортсмены, Тренеры, Тренировки) с автоматическим логированием всех операций и возможностью отмены действий.

## Выбранные таблицы

1. **Athlete** (Спортсмен) - центральная таблица
2. **Coach** (Тренер) - таблица с информацией о тренерах  
3. **Training** (Тренировка) - связующая таблица между спортсменами и тренерами

## Структура проекта

### SQL-скрипты

1. **1_create_tables.sql** - Создание основных таблиц и таблиц логирования
   - Таблицы: Athlete, Coach, Training
   - Таблицы аудита: Audit_Log, Undo_History
   - Индексы для оптимизации

2. **2_data_management_package.sql** - PL/SQL пакет для работы с данными
   - `insert_athlete()` - добавление спортсмена
   - `update_athlete()` - обновление спортсмена
   - `delete_athlete()` - удаление спортсмена
   - Аналогичные процедуры для Coach и Training

3. **3_audit_triggers.sql** - Триггеры для автоматического логирования
   - Триггеры на INSERT, UPDATE, DELETE для каждой из 3 таблиц
   - Автоматическое сохранение старых и новых значений в JSON
   - Запись информации о пользователе и времени операции
   - Дополнительные триггеры каскадного удаления связанных Training

4. **4_audit_management_package.sql** - PL/SQL пакет для управления логами
   - `view_audit_log()` - просмотр логов с фильтрацией
   - `undo_operation()` - отмена выбранной операции
   - `get_summary_report()` - сводный отчет с гибкой сортировкой

### Python-приложение

1. **app_backend.py** - Бэкенд приложения
   - Класс `DatabaseConnection` - управление подключением к Oracle
   - Классы `AthleteManager`, `CoachManager`, `TrainingManager` - работа с таблицами
   - Класс `AuditManager` - работа с логами и отменой операций

2. **app_gui.py** - GUI на основе tkinter
   - Интерфейс для подключения к БД
   - Отдельные вкладки для каждой таблицы
   - Вкладка Аудит для просмотра логов, отмены операций и получения отчетов
   - Выбор дат через календарь (`DateEntry`)
   - Автозаполнение форм по выбранной строке таблицы
   - Значения по умолчанию для ключевых полей (ID, даты, справочники)
   - Вывод сводного отчета прямо в интерфейс

3. **requirements.txt** - Зависимости Python

## Установка и запуск

- Python 3.11+
- oracledb для подключения к Oracle

## Использование приложения

### Подключение к БД
1. Заполните данные подключения (хост, пользователь, пароль)
2. Нажмите "Подключиться"
3. После успешного подключения кнопка станет неактивной

### Работа со спортсменами (вкладка "Спортсмены")
- **Добавить**: Заполните все поля и нажмите "Добавить"
- **Обновить**: Введите ID, заполните нужные поля и нажмите "Обновить"
- **Удалить**: Введите ID и нажмите "Удалить"
- **Обновить таблицу**: Отображает актуальный список

### Работа с тренерами (вкладка "Тренеры")
Аналогично спортсменам

### Работа с тренировками (вкладка "Тренировки")
Аналогично спортсменам

### Работа с логами (вкладка "Логирование и Аудит")

**Просмотр логов**
1. (Опционально) Введите диапазон дат и/или тип операции
2. Нажмите "Показать логи"
3. Логи будут отображены в текстовой области на вкладке

**Отмена операции**
1. Найдите ID лога в таблице
2. Введите ID в поле "ID лога для отмены"
3. Нажмите "Отменить операцию"
4. Система восстановит предыдущее состояние данных

**Сводный отчет**
1. Отметьте нужные флаги сортировки:
   - Флаг 1: Сортировка по названию сущности (первый приоритет)
   - Флаг 2: Сортировка по типу операции (второй приоритет)
   - Флаг 3: Сортировка по количеству (третий приоритет)
2. Нажмите "Показать отчет"
3. Отчет будет выведен в нижнюю область

## Архитектура системы логирования

### Таблица Audit_Log

```
log_id          - Уникальный ID записи лога
entity_name     - Название таблицы (ATHLETE, COACH, TRAINING)
entity_id       - ID записи в таблице
operation_type  - Тип операции (INSERT, UPDATE, DELETE)
operation_date  - Дата и время операции
old_data        - Старые значения (JSON формат)
new_data        - Новые значения (JSON формат)
user_name       - Пользователь, выполнивший операцию
is_reverted     - Флаг отмены операции (0 или 1)
```

### Таблица Undo_History

```
undo_id         - Уникальный ID записи об отмене
log_id          - Ссылка на Audit_Log
undo_date       - Дата и время отмены
undo_status     - Статус отмены (например: SUCCESS, ALREADY_EXISTS, ERROR: ...)
```

### Триггеры

Для каждой таблицы (Athlete, Coach, Training) созданы триггеры:

- **AFTER INSERT** - логирует добавление с новыми значениями
- **AFTER UPDATE** - логирует изменение со старыми и новыми значениями
- **BEFORE DELETE** - логирует удаление со старыми значениями

Дополнительно:
- **TRG_ATHLETE_CASCADE_DELETE_TRAINING** - удаляет связанные Training перед удалением Athlete
- **TRG_COACH_CASCADE_DELETE_TRAINING** - удаляет связанные Training перед удалением Coach

## Примеры использования SQL-пакетов

### Добавление спортсмена (из PL/SQL)
```sql
BEGIN
  data_management.insert_athlete(
    p_athlete_id => 1,
    p_full_name => 'Иван Петров',
    p_birth_date => TO_DATE('1990-05-15', 'YYYY-MM-DD'),
    p_gender => 'M',
    p_rank => 'Мастер спорта',
    p_club_id => 1
  );
END;
/
```

### Просмотр логов за период
```sql
BEGIN
  audit_management.view_audit_log(
    p_start_date => TRUNC(SYSDATE) - 7,
    p_end_date => TRUNC(SYSDATE),
    p_operation_type => 'UPDATE'
  );
END;
/
```

### Отмена операции
```sql
DECLARE
  v_status VARCHAR2(50);
BEGIN
  audit_management.undo_operation(p_log_id => 5, p_status => v_status);
  DBMS_OUTPUT.PUT_LINE('Статус: ' || v_status);
END;
/
```

### Получение сводного отчета (сортировка: сущность → операция → количество)
```sql
BEGIN
  audit_management.get_summary_report(
    p_sort_by_entity => 1,      -- Флаг 1: ДА
    p_sort_by_operation => 1,   -- Флаг 2: ДА
    p_sort_by_count => 1        -- Флаг 3: ДА
  );
END;
/
```

## Функциональность

✓ **Вставка данных** - Добавление новых записей в таблицы
✓ **Обновление данных** - Изменение существующих записей
✓ **Удаление данных** - Удаление записей из таблиц
✓ **Автоматическое логирование** - Все операции записываются в Audit_Log
✓ **Отмена операций** - Восстановление предыдущего состояния данных
✓ **Каскадное удаление связей** - Training удаляются при удалении Athlete/Coach
✓ **Фильтрация логов** - По дате и типу операции
✓ **Сводные отчеты** - С гибкой многоуровневой сортировкой
✓ **Графический интерфейс** - Удобная работа с данными и логами
✓ **Многопоточность** - Неблокирующие операции БД в GUI
✓ **Календарь дат** - Выбор дат через DateEntry
✓ **Отчет в GUI** - Сводка отображается прямо в интерфейсе
